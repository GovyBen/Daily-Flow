### I-005：KSP 在 ai:data 模块的兼容性问题

- **问题**：Koin KSP 编译器在 ai:data JVM 模块中持续生成零输出
- **决策**：禁用 ai:data 的 KSP 插件，改为手动 Koin 模块（显式 `single`/`factory`）
- **影响**：ai:data 新增工具类时需手动更新 Koin 模块定义；其他模块的 KSP 不受影响

## 2026-06-15

### I-001：Hermes 运行环境限制

- **问题**：Hermes Agent 运行在 Windows bash (git-bash/MSYS) 环境，无法直接执行 `.\gradlew.bat` + `-P` 参数的复杂命令行
- **决策**：Hermes 负责代码阅读、分析、搜索、编写和审查；构建验证由用户执行或 Hermes 尝试调用后反馈结果
- **影响**：代码提交前需用户或 CI 运行测试确认

### I-002：DF-012 Provider 契约测试未完成

- **问题**：P0.5 中的 DF-012（Provider 连接和能力测试）标记为 `[ ]`，是进入 P5 的前置条件
- **决策**：在 P7 前补齐 DF-012（注：P5 已在 DF-012 未完成的情况下推进，因其主要影响 Provider 兼容性保证，不阻塞提案模型开发）
- **影响**：P7 前需完成 DF-012 以确保所有 Provider 能力状态明确

### I-003：旧 alarm 表的兼容保留

- **问题**：DF-403 迁移时保留了旧 `alarms` 表，但 DF-405 将不再新增 alarm 行
- **决策**：DF-405 迁移后 `alarms` 表仅保留历史引用，不新增行，不物理删除表。后续版本统一清理
- **理由**：避免迁移期间数据丢失风险
